## 1. What this repo does

Playwright E2E tests against **Veltro Novo** — the Power Apps Code App that lives at
`https://apps.powerapps.com/play/e/31526b49-62a2-e0ac-84b7-c5b854521b49/app/801a1353-ed03-4c01-b215-c05c217eaf97`.

The setup is a fork of `microsoft/power-platform-playwright-samples` (Rush monorepo, Node 20.16 isolated via fnm) into `Power-Fox-Solutions/power-platform-playwright-samples`, cloned to `C:\Dev\clients\veltro-novo-e2e\`.

What's specific to Veltro Novo:
- **`pages/CodeAppPage.ts`** — new Page Object for Code Apps. Microsoft's toolkit ships `ModelDrivenAppPage` and `CanvasAppPage` but not one for Code Apps. Ours knows how to launch the app, switch into its iframe, click sidebar nav items, toggle theme/language, measure scroll overflow, and read identity from the sidebar footer.
- **`tests/veltro-novo/`** — Veltro Novo test suite. Phase 0: smoke tests for shell/routing/theme/overflow. Phases 1-6 will add CRUD + UX tests as features land.
- **Playwright project `veltro-novo-codeapp`** — added to `playwright.config.ts`. Uses the same MDA storage state (Code Apps share auth with MDAs because both run under `crm4.dynamics.com` cookies on the auth side).

What's NOT specific to Veltro Novo (we reused upstream):
- The `power-platform-playwright-toolkit` package (untouched).
- The auth flow via `playwright-ms-auth` library.
- The storage-state caching pattern (24h TTL).
- The global setup/teardown hooks.

## 2. Design decisions (so you don't second-guess them)

| Decision | Why |
|---|---|
| Single interactive user, no service principal | Yeriel pushed back on a hybrid SP+user approach. Code Apps require delegated user identity at runtime (the React app calls `getContext().user` and Dataverse queries go with the browser's user token). Testing "as a real user" is the goal anyway. Match LitoClean simplicity. |
| Dedicated account `e2e-veltro@powerfoxbi.com` | Tests should represent the typical user, not the super-admin. Yeriel's main account has Admin role — not representative. Dedicated account also lets us exempt MFA / Conditional Access without weakening Yeriel's own security posture. |
| Role `Veltro Novo User` (not Manager/Admin) | Same reasoning. If a flow needs Manager privileges, the test should assert that the User role gets blocked — that's data, not noise. |
| Storage state cached 24h | Standard `playwright-ms-auth` behaviour. Re-auth via the `auth-veltro.ps1` spawn-window pattern. Yeriel only sees an MFA prompt ~once/day. |
| Repo lives parallel at `C:\Dev\clients\veltro-novo-e2e\` | Same pattern as `litoclean-e2e`. Keeps Node 20.16 (Rush requirement) isolated from `veltro-novo` which uses modern Node. Avoids polluting the main app's lockfile. |
| Fork into `Power-Fox-Solutions` org | Survives Yeriel leaving the org. Allows future PRs to Microsoft upstream. |

### 5.3 Apply the MFA timeout patch

The `playwright-ms-auth` library has a hardcoded 5-second `waitForURL` timeout that breaks any tenant with MFA enabled (or even slow networks). The patch raises it to 180s.

**Why Yeriel has to do this, not me:** Claude's auto-mode classifier denies edits inside `node_modules` (it's a package-manager-managed directory, edits are dangerous). Yeriel runs the command manually once.

```powershell
# From C:\Dev\clients\veltro-novo-e2e
$FILE = (Get-ChildItem -Recurse -Filter authenticate.js -Path .\common\temp\node_modules\.pnpm\playwright-ms-auth@0.0.17_* | Select-Object -First 1).FullName
Copy-Item $FILE "$FILE.bak"
(Get-Content $FILE) -replace 'waitForURL\(targetUrl, \{ timeout: 5000 \}\)', 'waitForURL(targetUrl, { timeout: 180000 })' | Set-Content $FILE
Write-Host "Patched: $FILE"
```

**Long-term fix:** convert this to a proper `pnpm patch playwright-ms-auth` and commit the patch file. That survives `rush install`. Skip for now; do it after the first green smoke run.

```powershell

wt.exe -d "C:\Dev\clients\veltro-novo-e2e" powershell -NoExit -ExecutionPolicy Bypass -File ".\auth-veltro.ps1"
```

## 6. The smoke test suite

`tests/veltro-novo/smoke/shell.test.ts` covers:

| Test | What it asserts | Catches |
|---|---|---|
| boots through CodeAppGate and shows the sidebar | After `getContext()` resolves, sidebar has ≥8 nav items including Dashboard + Locations | CodeAppGate stuck on Loading; NAV_GROUPS regression |
| does NOT produce a body scrollbar on the empty Dashboard | `document.documentElement.scrollHeight <= window.innerHeight` on `#/` | The exact bug fixed in veltro-novo PR #27 |
| theme toggle flips the html class | After click, `<html>` class changes between `light`/`dark`/none | next-themes provider broken |
| language switcher persists choice | After click, `localStorage.i18n_language` is `en` or `da` | LanguageSwitcher disconnected |
| legacy hash routes still navigate | `#/locations`, `#/tasks`, `#/templates`, `#/inspections` set the hash | Router regression |

These 5 tests run in ~30s on a warm session. They're the regression suite for every PR touching layout/routing/providers.

## 8. Re-auth workflow (every 24h)

Symptom: `Authentication tokens have expired!` in any test run.

```powershell
# 1. Delete stale states (optional but recommended)
Remove-Item .\packages\e2e-tests\.playwright-ms-auth\state-*.json

# 2. Re-run the auth flow
wt.exe -d "C:\Dev\clients\veltro-novo-e2e" powershell -NoExit -ExecutionPolicy Bypass -File ".\auth-veltro.ps1"

# 3. Run tests
.\run-tests.ps1 smoke
```

If MFA is exempted for `e2e-veltro@`, no phone interaction needed. If not, approve on Authenticator within 3 minutes per step.

## 9. Operational notes

- **Cost:** zero infra; everything runs locally on your machine.
- **Time to first green smoke:** ~5 minutes once you have the account + .env filled.
- **Maintenance:** re-auth every 24h. The MFA patch wipes on `rush install` — convert to `pnpm patch` after the first green run for a permanent fix.
- **Watching tests run:** `HEADLESS=false` in `.env` (already set). Browser windows appear and you can see clicks.
- **Debugging a failing test:** `.\run-tests.ps1 <filter> -Debug` opens Playwright Inspector with step-through.
- **Visual report:** `.\run-tests.ps1 -Report` opens the last HTML report.
