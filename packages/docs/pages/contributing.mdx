# Contributing

Contributions are welcome! This page covers how to set up the development environment, contribute
code or documentation, and publish new releases.

<Callout type="info">
  By contributing you agree to the
  [Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct/).
</Callout>

## Development Setup

<Steps>

### Fork and clone

```bash
git clone https://github.com/microsoft/power-platform-playwright-samples.git
cd power-platform-playwright-samples
```

### Install Rush

```bash
npm install -g @microsoft/rush
```

### Install dependencies

```bash
rush install
```

### Build all packages

```bash
rush build
```

### Verify the sample tests compile

```bash
cd packages/e2e-tests
npx tsc --noEmit
```

</Steps>

## Making Changes

### Library changes (`packages/power-platform-playwright-toolkit/`)

1. Edit source files under `src/`
2. Export new APIs from `src/index.ts`
3. Add JSDoc/TSDoc comments to all public APIs
4. Rebuild: `rush build --to power-platform-playwright-toolkit`
5. Regenerate API docs: `cd packages/power-platform-playwright-toolkit && npm run docs`

### Adding a test

```bash
cd packages/e2e-tests
npx playwright test --project=mda     # Model-Driven
npx playwright test --project=canvas  # Canvas App
npx playwright test --project=gen-ux  # Gen UX
```

### Documentation changes (`packages/docs/`)

```bash
cd packages/docs
npm run dev      # Start local dev server at http://localhost:3000
```

## Submitting a Pull Request

1. Create a feature branch: `git checkout -b feat/my-feature`
2. Make your changes
3. Format code: `npm run format` (from repo root)
4. Lint: `rush lint`
5. Push and open a PR against `main`

PR titles should follow [Conventional Commits](https://www.conventionalcommits.org/):
`feat:`, `fix:`, `docs:`, `chore:`, `refactor:`

## Publishing the npm Package

<Callout type="warning">
  Only maintainers with npm publish rights can release a new version.
</Callout>

### Prerequisites

- npm account with publish rights to `power-platform-playwright-toolkit`
- `NPM_TOKEN` secret set in the GitHub repository settings (for automated publishing)

### Release process

<Steps>

### Update the version

Edit `packages/power-platform-playwright-toolkit/package.json` and bump the version following
[semantic versioning](https://semver.org/):

| Bump | When |
|------|------|
| **MAJOR** | Breaking API changes |
| **MINOR** | New features, backwards-compatible |
| **PATCH** | Bug fixes |

### Update CHANGELOG

Document the changes in `CHANGELOG.md`.

### Pre-release checklist

- [ ] All tests pass: `cd packages/e2e-tests && npx playwright test`
- [ ] Build succeeds: `rush build`
- [ ] API docs regenerated: `cd packages/power-platform-playwright-toolkit && npm run docs`
- [ ] Docs site builds: `cd packages/docs && npm run build:github`
- [ ] All changes committed

### Create a git tag and release

```bash
git commit -m "chore: release vX.Y.Z"
git tag vX.Y.Z
git push origin main --tags
```

Then create a GitHub Release from the tag — this triggers the automated npm publish workflow
(`.github/workflows/publish-npm.yml`).

### Manual publish (if needed)

```bash
rush build --to power-platform-playwright-toolkit
cd packages/power-platform-playwright-toolkit
npm publish
```

</Steps>

## Deploying Documentation

Documentation is automatically deployed to GitHub Pages when changes merge to `main`
in `packages/docs/**` or `packages/power-platform-playwright-toolkit/src/**`.

### Manual deployment

```bash
# Regenerate API reference
cd packages/power-platform-playwright-toolkit && npm run docs

# Build the static site
cd packages/docs && npm run build:github
```

The static output is in `packages/docs/out/`. GitHub Actions deploys it via
`.github/workflows/deploy-docs.yml`.

### Local preview

```bash
cd packages/docs
npm run dev     # Dev server with hot reload at http://localhost:3000
npm run build   # Production build (without GitHub base path)
npm run start   # Serve production build
```

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `rush install` fails | Run `rush purge && rush update --full` |
| TypeScript errors after changes | Run `rush build` to recompile all packages |
| npm publish fails — version exists | Bump the version number first |
| Docs build fails | Run `npm run generate:api` inside `packages/docs` to regenerate TypeDoc output |
| GitHub Pages not updating | Check the Actions tab — ensure `deploy-docs.yml` ran successfully |
