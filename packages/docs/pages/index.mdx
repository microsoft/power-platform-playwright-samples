# Power Platform Playwright Toolkit

<strong>
  Official Playwright automation framework for Microsoft Power Platform — open-sourced by Microsoft.
</strong>

A production-ready testing toolkit for Power Platform applications built on Playwright. Supports
**Canvas Apps**, **Model-Driven Apps**, **Custom Pages**, and **Gen UX** designer experiences with
first-class authentication, Page Object Models, FormContext API, and structured test patterns.

## Quick Start

<PackageInstall packages={['power-platform-playwright-toolkit', '@playwright/test']} />

```typescript
import { test } from '@playwright/test';
import { AppProvider, AppType, AppLaunchMode } from 'power-platform-playwright-toolkit';

test('my first Power Platform test', async ({ page, context }) => {
  const provider = new AppProvider(page, context);

  // Launch a Model-Driven App
  await provider.launch({
    app: 'Northwind Orders',
    type: AppType.ModelDriven,
    mode: AppLaunchMode.Play,
    skipMakerPortal: true,
    directUrl: process.env.MODEL_DRIVEN_APP_URL!,
  });

  const app = provider.getModelDrivenAppPage();
  await app.navigateToGridView('nwind_orders');
  await app.grid.waitForGridLoad();

  const count = await app.grid.getRowCount();
  expect(count).toBeGreaterThan(0);
});
```

## Why This Toolkit?

### Traditional Approach ❌

```typescript
// Brittle, hard-to-read selectors scattered everywhere
await page.locator('[data-control-name="Button1_1"]').click();
await page.locator('input[aria-label="Text input 3"]').fill('value');
await page.locator('[data-id="name.fieldControl-text-box-text"]').fill('Contoso');
```

### With This Toolkit ✅

```typescript
// Readable, maintainable, typed code
await provider.click({ name: 'SubmitButton' });
await app.fillTextInput('TextInputName', 'value');
await setEntityAttribute(page, 'name', 'Contoso');
```

## Key Capabilities

<Features>
  <Feature icon={<Target />} title="Page Object Models">
    Pre-built page objects for Canvas Apps, Model-Driven Apps, Custom Pages, and Gen UX with
    robust locators tuned for Power Platform's Fluent UI controls
  </Feature>
  <Feature icon={<Boxes />} title="Factory & Provider Patterns">
    `AppLauncherFactory` and `AppProvider` provide a consistent, type-safe API across all app
    types — no more per-app boilerplate
  </Feature>
  <Feature icon={<Database />} title="FormContext API">
    Read and write Dataverse form attributes programmatically, check dirty/valid state, and
    execute arbitrary Xrm JavaScript — all from Playwright test code
  </Feature>
  <Feature icon={<Zap />} title="Component Architecture">
    Composable `GridComponent`, `FormComponent`, and `CommandingComponent` separate concerns
    cleanly and make tests easier to read and maintain
  </Feature>
  <Feature icon={<Eye />} title="Gen UX Testing">
    Dedicated `GenUxPage` and UCI Preview iframe support for testing the Power Apps Gen UX
    designer experience
  </Feature>
  <Feature icon={<Shield />} title="Auth Token Validation">
    Built-in MSAL token expiry detection prevents cryptic mid-test authentication failures
  </Feature>
</Features>
