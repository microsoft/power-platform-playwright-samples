# Testing Canvas Apps

Canvas Apps are low-code applications built with visual controls. Each control has a **unique
name** set by the app maker — that name is your primary way to interact with it.

## Architecture Overview

<Features>
  <Feature icon={<Target />} title="Control-based Interactions">
    Click, fill, and read controls by their maker-assigned name
  </Feature>
  <Feature icon={<Palette />} title="Screen Navigation">

  </Feature>
  <Feature icon={<Zap />} title="Locator Utilities">

## Launching a Canvas App

<Callout type="info">
  Always use **`AppProvider`** as the entry point. Never instantiate `CanvasAppPage` directly.
</Callout>

```typescript
import { test } from '@playwright/test';
import {
  AppProvider,
  AppType,
  AppLaunchMode,
  CanvasAppPage,
} from 'power-platform-playwright-toolkit';

const APP_URL = process.env.CANVAS_APP_URL!;

test.describe('Canvas App Tests', () => {
  let app: CanvasAppPage;

  test.beforeEach(async ({ page, context }) => {
    const appProvider = new AppProvider(page, context);

    await appProvider.launch({
      app: 'My Canvas App',
      type: AppType.Canvas,
      mode: AppLaunchMode.Play,
      skipMakerPortal: true,
      directUrl: APP_URL,
    });

    app = appProvider.getCanvasAppPage();
  });
});
```

## Basic Control Interactions

### Click a control

```typescript
// Click by control name
await app.clickControl('ButtonSubmit');

// Click with options
await app.clickControl('ButtonSave', { timeout: 10000 });
```

### Fill text inputs

```typescript
// Fill by control name
await app.fillTextInput('TextInputName', 'Contoso Ltd');

// Clear then fill
await app.fillTextInput('TextInputEmail', 'test@contoso.com', { clear: true });

// Read the value back
const value = await app.getTextInputValue('TextInputName');
```

### Read control text

```typescript
// Get text content of a label or button
const labelText = await app.getControlText('LabelStatus');
const buttonLabel = await app.getControlText('ButtonSubmit');
```

### Check control visibility

```typescript
const isVisible = await app.isControlVisible('ButtonOptional');
if (isVisible) {
  await app.clickControl('ButtonOptional');
}
```

## Screens

Canvas Apps are organised into **screens**. Use these to verify navigation state:

```typescript
// Check which screen is currently active
const onHome = await app.isOnScreen('HomeScreen');
expect(onHome).toBe(true);

// Navigate programmatically (by clicking a button that triggers navigation)
await app.clickControl('ButtonViewOrders');
await page.waitForTimeout(1000);     // Allow screen transition to complete

const onOrders = await app.isOnScreen('OrdersScreen');
expect(onOrders).toBe(true);
```

## Galleries

Galleries display lists of data records in Canvas Apps:

```typescript
// Get total item count
const count = await app.getGalleryItemCount('GalleryOrders');

// Click a specific gallery item (0-indexed)
await app.clickGalleryItem('GalleryOrders', 0);

// Read all gallery items
const items = await app.getGalleryItems('GalleryOrders');
console.log(`Found ${items.length} items`);
```

## Dropdowns

```typescript
// Open and select a dropdown option
await app.clickControl('DropdownStatus');
await app.clickControl('DropdownStatus_Option_Active');

// Or with a dedicated method if available
await app.selectDropdownOption('DropdownStatus', 'Active');
```

## Low-Level Locator Utilities

For advanced scenarios where control names aren't available, use the locator utilities:

```typescript
import {
  getCanvasControlByName,
  getCanvasDataTestId,
  getCanvasScreenByName,
  CanvasControlType,
} from 'power-platform-playwright-toolkit';

// Locate a control by name (generates the data-control-name selector)
const submitBtn = page.locator(getCanvasControlByName('ButtonSubmit'));
await submitBtn.click();

// Locate by data-testid attribute
const input = page.locator(getCanvasDataTestId('order-number-input'));

// Locate a screen container
const homeScreen = page.locator(getCanvasScreenByName('HomeScreen'));
await expect(homeScreen).toBeVisible();
```

## Full Example: Order Submission Flow

```typescript
import { test, expect } from '@playwright/test';
import {
  AppProvider,
  AppType,
  AppLaunchMode,
  CanvasAppPage,
  generateUniqueTestId,
} from 'power-platform-playwright-toolkit';

const APP_URL = process.env.CANVAS_APP_URL!;

test.describe('Order Submission', () => {
  let app: CanvasAppPage;

  test.beforeEach(async ({ page, context }) => {
    const provider = new AppProvider(page, context);
    await provider.launch({
      app: 'Northwind Canvas',
      type: AppType.Canvas,
      mode: AppLaunchMode.Play,
      skipMakerPortal: true,
      directUrl: APP_URL,
    });
    app = provider.getCanvasAppPage();
  });

  test('should submit a new order', async ({ page }) => {
    // Start on home screen
    expect(await app.isOnScreen('HomeScreen')).toBe(true);

    // Navigate to new order screen
    await app.clickControl('ButtonNewOrder');
    await page.waitForTimeout(500);
    expect(await app.isOnScreen('NewOrderScreen')).toBe(true);

    // Fill in order details
    const orderId = generateUniqueTestId();
    await app.fillTextInput('TextInputOrderId', orderId);
    await app.fillTextInput('TextInputCustomer', 'Contoso Ltd');

    // Select status dropdown
    await app.clickControl('DropdownStatus');
    await app.clickControl('DropdownStatus_Option_Pending');

    // Submit
    await app.clickControl('ButtonSubmit');
    await page.waitForTimeout(1000);

    // Verify success
    const status = await app.getControlText('LabelSubmitStatus');
    expect(status).toContain('Order submitted');
  });

  test('should validate required fields', async ({ page }) => {
    await app.clickControl('ButtonNewOrder');
    await page.waitForTimeout(500);

    // Try submitting without filling required fields
    await app.clickControl('ButtonSubmit');

    const error = await app.getControlText('LabelValidationError');
    expect(error).toContain('required');
  });

  test('should navigate multi-screen flow', async ({ page }) => {
    // Home → Products → Details → Back
    expect(await app.isOnScreen('HomeScreen')).toBe(true);

    await app.clickControl('ButtonViewProducts');
    await page.waitForTimeout(500);
    expect(await app.isOnScreen('ProductsScreen')).toBe(true);

    const count = await app.getGalleryItemCount('GalleryProducts');
    expect(count).toBeGreaterThan(0);

    await app.clickGalleryItem('GalleryProducts', 0);
    await page.waitForTimeout(500);
    expect(await app.isOnScreen('ProductDetailScreen')).toBe(true);

    await app.clickControl('ButtonBack');
    await page.waitForTimeout(500);
    expect(await app.isOnScreen('ProductsScreen')).toBe(true);
  });
});
```

## Canvas App URL Builder

The toolkit provides utilities to build Canvas App URLs from environment variables:

```typescript
import {
  buildCanvasAppUrl,
  buildCanvasAppUrlFromEnv,
} from 'power-platform-playwright-toolkit';

// Build URL from components
const url = buildCanvasAppUrl({
  environmentId: process.env.POWER_APPS_ENVIRONMENT_ID!,
  appId: 'your-app-guid',
  tenantId: 'your-tenant-guid',
});

// Build from standard env vars
const urlFromEnv = buildCanvasAppUrlFromEnv();
```

## Best Practices

<Callout type="warning">
  Ask app makers to give controls **meaningful names** during development. Controls named `Button1`,
  `TextInput3` make tests fragile and hard to maintain.
</Callout>

<Tabs items={['Do this', 'Avoid this']}>
  <Tabs.Tab>
    ```typescript
    // Descriptive control names
    await app.clickControl('ButtonSubmitOrder');
    await app.fillTextInput('TextInputCustomerEmail', email);
    await app.clickGalleryItem('GalleryOrderHistory', 0);
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```typescript
    // Generic names — breaks when layout changes
    await app.clickControl('Button1');
    await app.fillTextInput('TextInput3', email);
    await app.clickGalleryItem('Gallery2', 0);
    ```
  </Tabs.Tab>
</Tabs>

| Practice | Reason |
|----------|--------|
| Wait for screen transitions after clicks | Canvas navigation is animated |
| Use `isControlVisible()` before conditional interactions | Controls may be hidden by app logic |
| Use `generateUniqueTestId()` for test data | Prevents collisions between parallel test runs |
| Avoid `page.waitForTimeout()` except for animations | Use `waitForElement()` or control visibility checks instead |
