# Authentication

Power Platform applications use Microsoft 365 (Entra ID) authentication. This toolkit uses
**`playwright-ms-auth`** to handle the full sign-in flow and persist the session as a storage
state file, so tests never need to re-authenticate on every run.

<Callout type="warning">
  **Security first:** Use dedicated test accounts with no MFA for automated runs. Never commit
  `.env` files or storage state JSON files to source control.
</Callout>

## How It Works

```
npm run auth              ─►  Headless browser opens → signs in to Microsoft 365
                                   ↓
                          Saves browser cookies + localStorage to state-{email}.json
                                   ↓
playwright.config.ts  ─►  Projects load that JSON as storageState
                                   ↓
Tests run             ─►  Already authenticated — no login prompt
```

The session remains valid for **approximately 1 hour** (MSAL token lifetime). When tokens expire,
re-run the auth script.

## Installation

`playwright-ms-auth` is a dependency of `power-platform-playwright-toolkit` — no separate install
needed if you have the toolkit installed.

## Authentication Scripts

Create `scripts/authenticate.ts` in your e2e package:

```typescript
// scripts/authenticate.ts
import { chromium } from '@playwright/test';
import {
  authenticateToMicrosoft,
  loadAuthConfig,
  getStorageStatePath,
} from 'power-platform-playwright-toolkit';
import * as path from 'path';
import * as dotenv from 'dotenv';

dotenv.config({ path: path.resolve(__dirname, '../.env') });

const isMda = process.argv.includes('--mda');
const isHeadful = process.argv.includes('--headful');

async function authenticate() {
  const config = loadAuthConfig();

  const browser = await chromium.launch({ headless: !isHeadful, channel: 'msedge' });
  const context = await browser.newContext();
  const page = await context.newPage();

  console.log(`Authenticating as: ${config.email}`);
  console.log(`Mode: ${isMda ? 'MDA' : 'Default'} | ${isHeadful ? 'Headful' : 'Headless'}`);

  await authenticateToMicrosoft(page, config);

  // Determine storage state path
  const baseStatePath = getStorageStatePath(config.email);
  const statePath = isMda
    ? path.join(path.dirname(baseStatePath), `state-mda-${config.email}.json`)
    : baseStatePath;

  await context.storageState({ path: statePath });
  await browser.close();

  console.log(`Auth state saved to: ${statePath}`);
}

authenticate().catch((err) => {
  console.error('Authentication failed:', err);
  process.exit(1);
});
```

Add scripts to `package.json`:

```json
{
  "scripts": {
    "auth": "tsx scripts/authenticate.ts",
    "auth:headful": "tsx scripts/authenticate.ts --headful",
    "auth:mda": "tsx scripts/authenticate.ts --mda",
    "auth:mda:headful": "tsx scripts/authenticate.ts --headful --mda"
  }
}
```

## Wiring Storage State into `playwright.config.ts`

```typescript
import { getStorageStatePath } from 'power-platform-playwright-toolkit';
import * as path from 'path';

export default defineConfig({
  projects: [
    {
      name: 'model-driven-app',
      testMatch: '**/mda/**/*.test.ts',
      use: {
        storageState: process.env.MS_AUTH_EMAIL
          ? path.join(
              path.dirname(getStorageStatePath(process.env.MS_AUTH_EMAIL)),
              `state-mda-${process.env.MS_AUTH_EMAIL}.json`
            )
          : undefined,
      },
    },
    {
      name: 'canvas-app',
      testMatch: '**/canvas/**/*.test.ts',
      use: {
        storageState: process.env.MS_AUTH_EMAIL
          ? getStorageStatePath(process.env.MS_AUTH_EMAIL)
          : undefined,
      },
    },
  ],
});
```

## Validating Auth State at Startup

The toolkit provides `ConfigHelper.checkStorageStateExpiration()` to detect expired MSAL tokens
before any tests run, giving a clear error instead of a cryptic authentication failure mid-test.

Create `utils/validate-auth-state.ts`:

```typescript
import { existsSync, unlinkSync } from 'fs';
import { getStorageStatePath, ConfigHelper } from 'power-platform-playwright-toolkit';
import * as path from 'path';

export function validateAuthState(email: string, isMda = false): void {
  const baseStatePath = getStorageStatePath(email);
  const statePath = isMda
    ? path.join(path.dirname(baseStatePath), `state-mda-${email}.json`)
    : baseStatePath;

  if (!existsSync(statePath)) {
    const cmd = isMda ? 'npm run auth:mda:headful' : 'npm run auth:headful';
    throw new Error(
      `Storage state not found: ${statePath}\nRun authentication first: ${cmd}`
    );
  }

  const check = ConfigHelper.checkStorageStateExpiration(statePath);
  if (check.expired) {
    const cmd = isMda ? 'npm run auth:mda:headful' : 'npm run auth:headful';
    const expiry = check.expiresOn ? new Date(check.expiresOn * 1000).toLocaleString() : 'unknown';

    // Remove stale state file so re-auth creates a clean one
    try {
      unlinkSync(statePath);
      console.log(`Removed expired state file: ${statePath}`);
    } catch {
      // Best-effort cleanup
    }

    throw new Error(
      `Auth token expired at ${expiry}.\nStale state file removed. Re-authenticate: ${cmd}`
    );
  }

  const minutes = check.expiresOn
    ? Math.floor((check.expiresOn - Date.now() / 1000) / 60)
    : null;
  console.log(
    `Auth valid${minutes !== null ? ` (expires in ${minutes} min)` : ''}: ${statePath}`
  );
}
```

Call it in `playwright.config.ts` before tests run:

```typescript
// At the top of playwright.config.ts (after dotenv.config)
const isMainProcess = !process.env.TEST_WORKER_INDEX && !process.env.PLAYWRIGHT_WORKER;
if (!process.argv.includes('list-files') && !process.env.CI && isMainProcess) {
  validateAuthState(process.env.MS_AUTH_EMAIL!, /* isMda */ false);
}
```

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| `Storage state not found` | Auth script not run | Run `npm run auth:headful` |
| `Auth token expired` | MSAL token > 1 hour old | Stale state file is auto-deleted — re-run `npm run auth:headful` |
| Tests show login page | Wrong storage state path | Check `MS_AUTH_EMAIL` env var is set |
| MFA prompt during auth | Account has MFA enabled | Use headful mode; disable MFA on test account |
| `Permission denied` to app | Account lacks app access | Grant the test account access in Power Platform Admin |

<Callout type="info">
  When tokens expire during a long test run, delete `state-*.json` files and re-authenticate:
  ```bash
  rm state-*.json && npm run auth:headful && npm run auth:mda:headful
  ```
</Callout>
